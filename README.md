# Magento Amasty Home Hreflang Fix

Production-safe Magento 2 fix for incorrect HTML hreflang URLs generated by Amasty on multilingual stores.

## Problem

On multilingual Magento stores, Amasty may output homepage hreflang URLs using the CMS page identifier instead of the real store base URL.

Example:

Wrong:
- `/it/athlete2-home-fi`
- `/en/athlete2-home-fi`

Correct:
- `/it/`
- `/en/`

## What this module does

This custom Magento 2 module adds a plugin on:

`Amasty\XmlsitemapHtmlHreflang\Model\AddHreflangToPageConfig::execute`

When the current CMS page is the configured homepage of the current store, the module replaces hreflang URLs with the correct store base URLs.

## Scope

Current scope:

- CMS homepage hreflang fix
- Vendor-safe implementation via plugin
- No core edits
- No vendor edits
